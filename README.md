This project demonstrates how Excel can be used as a lightweight Project Control System for small construction and service businesses.
The solution was designed using business analysis best practices, focusing on requirements, process improvement, data structure, and reporting.

Business Problem

Small contractors often manage projects using disconnected spreadsheets, emails, and paper invoices, leading to:

missed invoice follow-ups

unclear payment and holdback status

limited visibility into project profitability

Objective

Design a centralized Excel-based system to:

track projects, invoices, payments, expenses, and change orders

calculate real-time KPIs (progress %, paid %, remaining balance)

support better financial and operational decision-making

My Role (Business Analyst)

Defined business objectives and scope

Documented functional and non-functional requirements

Designed data structure and validation rules

Modeled current-state and future-state workflows

Built a working Excel MVP with reporting dashboards

Key Features
